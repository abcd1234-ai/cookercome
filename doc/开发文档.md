# 项目架构与技术栈

项目采用**前后端分离架构**，分为**用户端**、**商家端**、**管理员端**三个端，后端提供统一的 API 接口。

## 核心技术栈

| 层级 | 技术选型 |
|------|----------|
| **后端** | Spring Boot + MyBatis-Plus + MySQL + Redis |
| **前端** | 用户/商家端（UniApp / 微信小程序）、管理员端（Vue + Element UI） |
| **第三方服务** | 地图定位（高德/百度地图 API）、支付（微信/支付宝支付）、图片存储（OSS） |

## 3.2 用户端（客户）功能模块

| 模块 | 子功能点 | 接口设计要点 |
|------|----------|--------------|
| **账号管理** | 1. 手机号注册 / 验证码登录<br>2. 个人信息修改（头像、姓名）<br>3. 密码修改 | 1. 登录接口返回 JWT 令牌<br>2. 信息修改需验证用户身份<br>3. 密码加密存储（BCrypt） |
| **地址管理** | 1. 新增 / 编辑 / 删除地址<br>2. 设为默认地址<br>3. 定位自动填充地址 | 1. 地址保存时同步解析经纬度<br>2. 默认地址仅允许一个 |
| **厨师筛选** | 1. 附近厨师推荐（按距离排序）<br>2. 按菜系 / 价格 / 评分筛选 | 1. 基于 Haversine 公式计算距离<br>2. 仅展示审核通过、可接单的厨师 |
| **厨师详情** | 1. 厨师基础信息展示<br>2. 特色菜品查看<br>3. 用户评价查看 | 1. 关联查询 `t_chef_info`、`t_chef_dish`、`t_evaluation` 表 |
| **订单管理** | 1. 提交预约订单<br>2. 订单支付<br>3. 订单列表 / 详情<br>4. 取消订单 | 1. 订单编号生成规则：时间戳 + 随机数<br>2. 支付回调更新订单状态<br>3. 取消订单校验状态 |
| **评价管理** | 1. 服务完成后评价厨师<br>2. 查看自己的评价记录 | 1. 仅服务完成的订单可评价<br>2. 评价支持上传多张图片 |

## 3.3 商家端（厨师）功能模块

| 模块 | 子功能点 | 接口设计要点 |
|------|----------|--------------|
| **账号与资质** | 1. 手机号登录<br>2. 提交资质审核（身份证照片、烹饪年限）<br>3. 查看审核状态 | 1. 仅 `role=1` 的用户可访问<br>2. 审核状态变更推送通知<br>3. 身份证照片上传至 OSS |
| **信息管理** | 1. 厨师简介修改<br>2. 服务半径 / 定价修改<br>3. 服务状态切换（可接单 / 休息） | 1. 服务状态变更实时生效<br>2. 定价修改需验证合理性（≥0） |
| **菜系与菜品** | 1. 新增 / 删除擅长菜系<br>2. 标记主打菜系<br>3. 菜品增删改查 | 1. 菜系关联 `t_cuisine` 字典表<br>2. 菜品图片支持预览 / 上传 |
| **订单处理** | 1. 待接单列表<br>2. 接单 / 拒单<br>3. 确认服务开始 / 结束<br>4. 收入查看 | 1. 拒单需填写原因<br>2. 服务完成后触发评价提醒<br>3. 收入统计按订单状态筛选 |
| **评价管理** | 1. 查看用户评价<br>2. 回复评价（可选） | 1. 评价按时间倒序展示<br>2. 回复内容审核（可选） |

## 3.4 管理员端功能模块

| 模块 | 子功能点 | 接口设计要点 |
|------|----------|--------------|
| **账号管控** | 1. 查看所有用户<br>2. 禁用 / 启用用户账号<br>3. 重置用户密码 | 1. 仅 `role=2` 的用户可访问<br>2. 操作记录日志（谁、何时、操作内容） |
| **厨师审核** | 1. 待审核厨师列表<br>2. 审核通过 / 拒绝（填写备注）<br>3. 复审已审核厨师 | 1. 审核操作更新 `t_chef_info.audit_status`<br>2. 拒绝原因需明确 |
| **基础数据管理** | 1. 菜系增删改查 / 排序<br>2. 角色字典管理（扩展用） | 1. 菜系禁用后，关联菜品不可见<br>2. 排序值影响前端展示顺序 |
| **订单管控** | 1. 全量订单查询<br>2. 按状态 / 时间筛选<br>3. 强制取消异常订单 | 1. 强制取消订单需校验权限<br>2. 取消后同步处理退款（如有） |
| **数据统计** | 1. 核心数据看板（用户数、厨师数、订单数）<br>2. 厨师 / 订单趋势分析 | 1. 数据缓存（Redis），避免高频查询 DB<br>2. 支持导出 Excel |
